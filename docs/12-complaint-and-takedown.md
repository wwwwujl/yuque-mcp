# 投诉与下架流程（Complaint and Takedown）

本项目提供公开的投诉与下架处理流程，用于处理版权、滥用、安全与合规相关问题。

## 可受理范围

- 版权或知识产权投诉
- 恶意使用、滥用或误导性使用反馈
- 隐私或敏感信息泄露线索
- 其他需要临时下架或限制分发的合规问题

## 提交方式

- 优先通过 GitHub Issue 模板提交：
  - `Complaint / Takedown request`
- 如果内容涉及敏感信息，不要在公开 issue 中贴出完整证据；只提供最小必要信息并请求私下沟通。

## 投诉内容建议

- 投诉类型（版权/滥用/隐私/其他）
- 受影响对象（仓库路径、版本号、发布链接）
- 事实描述与证据链接
- 诉求（移除内容、停止分发、澄清说明等）
- 联系方式（用于后续沟通）

## 处理时效（目标）

- 48 小时内确认收到
- 7 天内给出初步处理结论
- 紧急高风险问题可先采取临时措施，再补充完整结论

## 可能的处理动作

- 文档修订或补充声明
- 修复并发布新版本
- 对 npm 包进行 `deprecate` 警示
- 在必要情况下停止继续分发新版本，或按平台规则执行下架动作

## 备注

- 恶意或明显缺乏事实依据的投诉可不予处理。
- 本流程不替代司法或行政程序；如有法律要求，以法律程序为准。
